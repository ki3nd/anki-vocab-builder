Build a sentence structures reference table at `tables/sentence_structures.md`.

Source files:
- `output/sentence_structures.md` — structure list (one per line, e.g. `as ... as possible`)
- `reading.md` — the original reading text (used for context when creating examples)

This skill does **NOT** call any API. All definitions, examples, and tags are generated by you based on your own knowledge.

---

## Setup

1. Read `output/sentence_structures.md` to get the full list.
2. Read `reading.md` to understand the topic and context.
3. Run `mkdir -p tables` to ensure the output folder exists.
4. If `tables/sentence_structures.md` does not exist yet, create it and write this exact header as the first two lines:

```
| structure | meaning | examples | equivalent | tags |
|-----------|---------|----------|------------|------|
```

---

## Processing — do this for EVERY structure, one at a time

For each structure in the list, follow these steps **in order**, then **immediately append** the result row to `tables/sentence_structures.md` before moving to the next one.

### Step 1 — Write the meaning

Write a brief, clear explanation of:
- What this structure means
- When and why to use it

Keep it to 1–2 sentences. Write in English.

### Step 2 — Create exactly 4 examples

Write 4 example sentences using this structure:
- 2 examples should be relevant to the topic of `reading.md` (e.g. renting an apartment, deposits, applications, maintenance)
- 2 examples should be on general everyday topics (to show the structure is versatile)
- All 4 must clearly demonstrate the structure in use
- Be natural and varied

### Step 3 — Write equivalent structures

List 1–3 alternative structures or expressions that convey the same or very similar meaning.

For example:
- `in order to + V` → equivalents: `so as to + V`, `to + V (purpose)`
- `as ... as possible` → equivalents: `as ... as one can`, `the most ... possible`
- `be required to + V` → equivalents: `must + V`, `have to + V`, `be obliged to + V`

If no clear equivalent exists, leave empty.

### Step 4 — Add tags

Tag the structure with relevant usage context. Choose from:
- Formality: `formal`, `informal`, `neutral`
- Mode: `spoken`, `written`, `both`
- Exam relevance: `IELTS writing`, `IELTS speaking`, `TOEIC`, `Cambridge`
- Any other useful label

### Step 5 — Build the row

Combine all **4 examples** into the `examples` cell, separated by ` · `
Combine equivalents into the `equivalent` cell, separated by `, `
Combine tags into the `tags` cell, separated by `, `

Escape any `|` characters in cell content with `\|`. Replace newlines in cell content with a space.

Write the row in this exact format:
```
| structure | meaning | examples | equivalent | tags |
```

### Step 6 — Append immediately

Append the completed row to `tables/sentence_structures.md` right away. Do not wait until all structures are processed.

---

## Important rules

- Process structures **in order** as they appear in `output/sentence_structures.md`
- Keep all cell content on a **single line** — no line breaks inside a cell
- Do not add extra blank lines between rows
- All content is self-generated — no API calls needed
